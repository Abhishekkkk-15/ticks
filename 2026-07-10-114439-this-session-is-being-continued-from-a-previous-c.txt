     Custom window chrome, minimal scrollbar, zoom shortcuts, and a 
     workspace-navigation bug fix

     Context

     Four independent asks bundled together:
     1. Custom-drawn minimize/maximize/close buttons instead of the native
     Windows/Linux title bar (macOS keeps its native traffic lights —
     hiddenInset already gives that for free and there's no reason to
     touch it).
     2. A themed, minimal scrollbar instead of the OS-default one.
     3. Ctrl/Cmd + =/-/0 zoom shortcuts.
     4. A real bug: App.tsx's effect that auto-selects default_workspace_id
     on startup re-fires every time selectedWorkspace becomes null —
     including when the user deliberately clicks the sidebar's back arrow
     (NoteList.tsx's onBack → Sidebar.tsx's onSelectWorkspace(null))
     — so with a default workspace set, "back to all workspaces" is
     immediately overridden back into that workspace. Confirmed by reading
     App.tsx:36-49: the effect's guard is only !selectedWorkspace, with
     no way to distinguish "never selected yet" from "user backed out."

     Approach

     1. Bug fix (do this first, it's the smallest and most isolated)

     In frontend/src/renderer/src/App.tsx, add a useRef(false) flag
     (autoSelectedDefaultRef) that gets set to true the first time the
     effect auto-selects the default workspace, and gate the effect's body on
     !autoSelectedDefaultRef.current too. This preserves "load into my
     default workspace on launch" while letting the back arrow actually work
     afterward.

     2. Custom window controls (Windows/Linux only)

     - frontend/src/main/index.ts: change the BrowserWindow options so
     the native frame is only kept on macOS:
     frame: process.platform === 'darwin', and only set titleBarStyle: 
     'hiddenInset' on darwin (omit it elsewhere — it's meaningless without
     a frame). Add three ipcMain.handle/ipcMain.on entries:
     window:minimize (win.minimize()), window:toggle-maximize
     (win.isMaximized() ? win.unmaximize() : win.maximize()), and
     window:close (win.close()). Forward maximize state changes to the
     renderer so the button icon can flip: win.on('maximize', () => 
     win.webContents.send('window:maximized-changed', true)) and the same
     for 'unmaximize' with false. Also expose the platform so the
     renderer knows whether to render the bar at all — cheapest way is an
     ipcMain.handle('platform:get', () => process.platform) or just
     reading process.platform isn't available in the sandboxed renderer,
     so pass it through preload as a plain exposed value (not a function,
     it's static).
     - frontend/src/preload/index.ts + index.d.ts: extend the existing
     api object (same pattern as notifySettingsUpdated/onCaptureText)
     with platform: process.platform, windowControls: { minimize, 
     toggleMaximize, close, onMaximizedChange(callback): () => void }, following
     the exact
     ipcRenderer.on(...) / return unsubscribe pattern already used by
     onCaptureText.
     - New frontend/src/renderer/src/components/layout/TitleBar.tsx:
     renders null when window.api.platform === 'darwin'. Otherwise a
     slim (32px) flex row: app icon + "AI Learning Workspace" label in a
     drag region (style={{ WebkitAppRegion: 'drag' }}), then three icon
     buttons (Minus, Square/Copy for maximize/restore toggle based on
     state from onMaximizedChange, X from lucide-react) each with
     WebkitAppRegion: 'no-drag', calling the new preload methods.
     Double-click on the drag region should also call toggleMaximize (a
     onDoubleClick handler), matching standard OS title-bar behavior.
     - App.tsx: wrap the existing root <AppShell> return with a new
     outer flex h-screen flex-col container: <TitleBar /> on top, then
     <div className="min-h-0 flex-1"><AppShell>...</AppShell></div>.
     Change AppShell.tsx's root div from h-screen to h-full since it's
     no longer the outermost sized element.

     3. Minimal scrollbar

     Add plain CSS to frontend/src/renderer/src/assets/main.css (global,
     not layered — nothing else targets ::-webkit-scrollbar):
     thin width (~8-10px), transparent track, rounded thumb using the
     existing themed var(--color-neutral-700) (so it automatically follows
     Dark/Light/Warm-Cozy), background-clip: padding-box with a transparent
     border for a little inset padding, and a :hover state one shade
     lighter. This is Chromium/Electron so ::-webkit-scrollbar-* alone is
     sufficient (no Firefox scrollbar-color fallback needed).

     needed. In createWindow() in frontend/src/main/index.ts, after

     (input.control || input.meta) and input.key is '='/'+' → zoom in,
     '-' → zoom out, '0' → reset, each via webContents.getZoomLevel() /
     setZoomLevel(), clamped to a sane range (e.g. ±4, matching Chromium's
     ~0.42x–2.4x at the default 1.2^level formula), calling
     event.preventDefault() so it doesn't also trigger any browser-default
     behavior.

     Mini-tray: always-on-top, editor-only quick-note window

     Context

     The user wants a small, always-on-top "mini-tray" window — editor only,
     no sidebar/tabs/toolbar — toggled by a global keyboard shortcut so they
     can jump straight into typing from anywhere, then dismiss it just as
     fast. Target size is exact: 353×743px, taken directly from their
     reference screenshot. This mirrors the existing global-capture shortcut
     feature (registerGlobalCapture() in frontend/src/main/index.ts) but
     opens a real editor instead of just grabbing clipboard text.

     Confirmed via code survey: useNoteEditor(workspaceId, noteId) and
     MarkdownEditor (frontend/src/renderer/src/features/editor/) are
     already fully self-contained and chrome-free — they don't need
     EditorView's mode-toggle bar around them, so the mini window can reuse
     them directly instead of duplicating editor logic. The active note's
     identity lives only in App.tsx's activeTab ({workspaceId, note})
     state, which doesn't automatically cross into a second BrowserWindow's
     separate renderer process, so a small IPC bridge is needed to mirror it.

     Approach

     One extra always-on-top BrowserWindow, reusing the same renderer
     bundle via a ?mode=mini query param (no electron-vite config changes
     needed — src/renderer/src/main.tsx just branches on it), rather than a
     second Vite entry point or a mode-switch of the main window itself. The
     mini window is created lazily on first shortcut press, then only
     shown/hidden afterward (never destroyed), so scroll position/undo history
     persist across toggles.

     1. Main process (frontend/src/main/index.ts)

     - createMiniWindow(): width: 353, height: 743, resizable: false, frame: 
     false, transparent: true, alwaysOnTop: true, skipTaskbar: true, show: 
     false, positioned at the bottom-right of the primary display's
     work area (with a small margin). Always frameless/transparent on every
     platform — unlike the main window, this one has no native chrome on
     any OS, matching "only show the editor." Loads the same
     preload/index.js and either ELECTRON_RENDERER_URL + '?mode=mini'
     (dev) or loadFile(indexPath, { query: { mode: 'mini' } }) (prod).
     - toggleMiniTray(): create the window on first call; otherwise if
     visible, hide() it — if hidden, show() + focus() it, send the
     current active note over IPC, and ping the renderer to focus the
     editor. This single action covers both "focus shortcut" and "get
     out/in of mini mode" from the request.
     - New shortcut, following the exact getGlobalCaptureShortcut/
     mapShortcutToAccelerator/activeCaptureShortcut pattern already used
     for global capture: keyboard_shortcuts.mini_tray_toggle (default
     Ctrl+Alt+Shift+M), registered/re-registered in the same places
     (settings:updated handler + startup).
     - New IPC: ipcMain.on('active-note:changed', (e, payload) => { 
     lastActiveNote = payload; 
     miniWindow?.webContents.send('mini:active-note-changed', payload) })
     and ipcMain.handle('active-note:get', () => lastActiveNote), where
     lastActiveNote: { workspaceId: string; noteId: string } | null is a
     module-level var (same style as activeCaptureShortcut).

     2. Preload (index.ts + index.d.ts)
